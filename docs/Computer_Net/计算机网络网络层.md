# 数据平面

## 网络层概述

网络层将来自运输层的报文段封装成数据报, 或从数据报中提取报文段交付给运输层. 数据平面的作用是从输入链路向输出链路转发数据; 控制平面的作用是协调所有本地路由器的转发动作. 路由器没有网络层之上的部分.

- **转发**: 将分组从输入链路移动到适当的输出链路, 这是数据平面的唯一功能.
- **路由选择**: 决定分组所采用的路由或路径, 计算路径的算法称为 **路由选择算法**, 这在控制平面中实现.
- **转发表**: 路由器检查分组的首部字段, 路由器通过查转发表决定分组将被转发到哪个输出链路接口.

**链路层交换机** 是根据链路层帧中的字段做转发决定, 路由器使用过网络层数据报中的首部字段做转发决定, 所以路由器属于网络层设备, 而交换机属于链路层设备.

## 路由器

![1699375304256](/img/计算机网络网络层/1699375304256.png)

- 输入端口: 输入端口中将会进行查找功能, 通过查找路由表确定数据将要转发到哪一个输出端口.
- 交换结构: 将输入端口连接到输出端口, 这在后面会看到有多种形式.
- 输出端口: 存储从交换结构收到的数据, 并执行传输功能.
- 路由选择处理器: 执行控制平面的功能, 即路由选择协议.

### 输入端口处理

![1699700235025](/img/计算机网络网络层/1699700235025.png)

路由器使用 **最长前缀匹配规则** 选择输出端口, 根据地址在转发表中最长匹配的前缀对应的端口转发数据.

### 交换

![1699700418495](/img/计算机网络网络层/1699700418495.png)

- 经内存的交换: 一次只能传输一个分组, 因为共享系统总线一次只能进行一个内存读/写操作.
- 经总线的交换: 因为使用总线的原因, 所以一次只能传输一个分组.
- 经互联网络的交换: 可以在不冲突的前提下同时交换多个分组.

### 输出端口处理

![1699700691630](/img/计算机网络网络层/1699700691630.png)

### 排队现象

1. 输入排队: 如果交换结构不能很快的(相对于输入线路的速度)将分组传送, 就会在输入端口处产生排队. 对于互联网络交换结构, 如果因为某一个输入分组的传输妨碍了其他到空闲输出端口的输入分组, 被称为线路前部(HOL, head of the line)阻塞.
2. 输出排队: 当交换结构的速度大于输出端口的速度时, 会在输出端口处产生排队. 如果没有足够的内存时, 就要考虑丢弃一些分组, 一种方法是 **弃尾**, 另一种就是删除一个已排队的分组. 有时候, 在队列缓冲填满之前就丢弃一个分组, 称为 **主动队列管理(AQM)**, **随机早期检测(RED)** 是一种广泛使用的算法.

### 分组调度

解决排队的分组如何经输出链路传输的问题.

1. 先进先出
2. 优先权排队, 同一优先权下采用先进先出, 一般是非抢占式优先权排队
3. 循环排队, 分组会像优先权那样排队, 每一种都对应一个队列, **保持工作排队** 规则将不允许链路空闲, 于是就循环检查队列, 一旦其对应的输出链路空闲就传输它.
4. 加权公平排序, 为每一个输出队列分配一个权重, 保证其至少占有权重百分比的输出速率.

## 网际协议

### IPv4 报文

![1699711303881](/img/计算机网络网络层/1699711303881.png)

- 版本号: 用于区分数据报文的格式.
- 首部长度: 因为数据报存在可选的数据, 所以报文长度是可变的.
- 服务类型: 区分不同类型的数据报, 比如要求低时延, 可靠性的数据报
- 数据报长度: IP数据报的总字节长度, 包括首部
- 标识, 标志, 片偏移: 与 IP 分片有关
- 寿命: Time-To-Live(TTL) 是数据报在网络中经过路由器的最大跳数, 当TTL为0时路由器就丢弃该数据报.
- 协议: 指明上层协议, 比如TCP是6, UDP是17
- 首部校验和: 数据报按照 2 字节求和并取反码, 首部校验和字段初始时为 0
- 源/目的IP: 如名字
- 选项: 可以进行首部扩展, 不过 IPv6 已经删除此项
- 数据(有效载荷): 要交付给运输层的报文段(TCP, UDP, ICMP等)

### IPv4数据分片

一个链路层帧能传输的最大数据为 **最大传送单元** 通常为 1500 字节. 当一个IP数据报大小超过 1500 字节时, 就需要对数据报分成多个不超过 1500 字节的数据报.

![1699729324366](/img/计算机网络网络层/1699729324366.png)

如上图, 一个20字节头部+3000字节数据的 IP 报文, 被分为了三部分, data1 大小为 1480 字节, data2 为 1480 字节, data3 为 40 字节, 再加上 3 个首部, 那么原来大小为 3020 字节的报文就被分为了 3 个总大小为 3060 字节的报文.

**标识**, 用于区分一组分片, 原来属于同一个数据报的分片拥有相同的标识.

**标志**, 除了最后一个分片的标志为0外, 其他都为1, 这是为了告知接收方分片的结束.

**片偏移**, 是分片中的数据在原来数据报的有效载荷中的偏移字节除以8, 比如上边的三个分片的片偏移分别为 0, 185, 370.

### IPv4编址

#### IPv4地址形式

形如 a.b.c.d/x 的地址, 其中 a,b,c,d 都是 8 位的数字(0-255), x 是子网掩码, 比如子网掩码是 255.255.255.0 前 24 位是 1, 那么对应子网的 IP 就是 a.b.c.d/24. 这里要会求一个子网中最大容许的 IP 数.

#### 动态获取IP

**动态主机配置协议 (DHCP)** 允许主机自动获取一个 IP 地址.

- DHCP服务器发现: 一个新到达的主机通过 DHCP 发现报文发现一个要与其交互的 DHCP 服务器. DHCP 发现报文中目的 IP 为广播地址 255.255.255.255, 并且使用'本主机'源 IP 地址 0.0.0.0.
- DHCP 服务器提供: DHCP 服务器接收到一个发现报文时, 用 DHCP 提供报文向客户做出相应, 该报文向子网所有节点广播(广播地址为 255.255.255.255), 因为网络中可能存在多个 DHCP 服务器, 主机会从中进行选择. 提供报文中包含发现报文的事务 ID , 向客户推荐的 IP, 网络掩码, IP 租用期(通常为几个小时或者几天)
- DHCP 请求: 客户向选中的服务器发送 DHCP 请求报文进行响应.
- DHCP ACK: 服务器用 DHCP ACK 报文对请求报文进行响应, 客户接收到 ACK 后即可使用服务器刚才分配的 IP

### NAT

网络地址转换, 使得多个主机在互联网中共用一个 IP, 解决 IPv4 IP 不够的问题. 主要通过在 NAT 路由器上设置 NAT 转换表来实现.

## 路由选择算法

### 分类

分类一

- 集中式选择算法: 用完整, 全局的网络知识来计算最优路径, 被称为 **链路状态(LS)算法**
- 分散式选择算法: 节点没有所有网络链路的开销, 通过迭代的方式逐渐计算出最优路径, 称为 **距离向量(DV)算法**

分类二

- 静态路由选择算法: 路由随时间变化很慢, 由人工设置
- 动态路由选择算法: 随着网络负载或拓扑变化而自动改变路由选择路径

分类三

- 负载敏感算法: 链路开销会动态的反映出底层链路的当前拥塞水平
- 负载迟钝算法: 某条链路的开销不明确的反映出当前(最近)的拥塞水平

### LS 算法

通过让每个节点广播链路状态分组来实现网络中的节点对全局的认知. 实践中(比如OSPF协议)经常是由 **链路状态广播** 算法实现, 当节点已知网络状态后, 通过 Dijkstra 算法计算最短路径(教材246页). 计算完成后, 每个节点存储一个目的地-链路(下一跳)的转发表.

该算法是负载敏感型算法, 可能产生振荡现象.

### DV 算法

- 分布式的: 每个节点都需要从相邻的一个或多个节点接收信息
- 迭代的: 此过程会一直持续到邻居之间没有信息可交互, 是可以自我终止的
- 异步的: 不要求所有的节点之间步伐一致的执行操作

算法过程:

- 对于每个邻居 v, 从 x 到直接相连的邻居的开销为 c(x, v)
- 节点 x 的距离向量, 即 $D_x = [D_X(y): y \in N]$, N 为 x 的可达目的地集合
- 节点 x 的邻居的距离向量为 $D_v$
- 当节点接收到邻居的距离向量时, 使用 Bellman-Ford 方程 $D_x(y) = min_v\{c(x, v)+D_c(y) \} for each v in N$ 更新当前节点的距离向量
- 重复上述过程, 直到收敛(节点的距离向量不变)

路由选择环路现象:

教材252页

毒性逆转:

为了避免路由选择环路现象, 做法: 如果 x 通过 y 到达 z, 那么 x 告知 y 到 z 的距离是无穷. 但是当节点超过3个时, 该方法无法解决路由环路.

### LS 对比 DV

- 报文复杂性: LS 要求每次都发送 $O(N*E)$ 个报文, 而 DV 只在最低开销路径发生改变时才传播.
- 收敛速度: LS 是 $O(N^2)$ 的算法, DV 收敛速度慢且会遇到无穷计数问题.
- 健壮性: DV 算法会将一个不正确的节点计算值扩散到整个网络, LS 只会发送到相邻节点

## OSPF - 开放最短路优先

优点:

- 安全: 能够鉴别OSPF路由器之间的交换, 只有受信任的路由器能参与一个AS(自治系统)内的OSPF协议.
- 多条相同开销的路径: 可以使用多条开销相同的路径, 将流量分散到这些路径中.
- 支持单播与多播路由选择: 多播OSPF(MOSPF)提供多播支持.
- 支持在单个AS中的层次结构: 一个OSPF自治系统能够层次化的配置多个区域, 每个区域运行自己的OSPF链路状态路由选择协议.

## BGP - 边界网关协议

BGP 是自治系统之间的路由选择协议.

### BGP的作用

在每一个AS内部的转发由AS内部的路由协议决定, 而转发到该AS外部的目的地就需要通过BGP协议. BGP中的一个目的地采用前缀的形式, 比如 138.16.68/22. BGP提供了完成以下任务的手段:

- 从邻居AS获得前缀的可达信息: 允许每个AS向因特网其余部分告知自己的存在
- 确定到该前缀的"最好"路由: 在多条路径中, 路由器在本地运行一个BGP路由选择过程.

### 通告BGP路由信息

对于下图中的网络, 红圈圈出来的是每个AS的网关路由器, 其余的是各自AS的内部路由器. 在 BGP 中，每对路由器通过使用 179 端口的半永久TCP 连接交换路由选择信息. 每条直接连接以及所有通过该连接发送的 BGP 报文，称为BGP 连接(BGP connection). 跨越两个AS的BGP链接称为 **外部BGP(eBGP)** 链接, 同一个AS中的两台路由器之间的BGP会话称为 **内部BGP(iBGP)** 链接.

![1700382752448](/img/计算机网络网络层/1700382752448.png)

例如, 现在AS1要知道如何到达3c前缀. 网关路由器3a通过eBGP向网关路由器2c发送eBGP报文"AS3 3c", 然后2a告知AS2中的所有路由器, 发送iBGP报文"AS3 3c", 网关路由器2a再向AS1的网关路由器1c发送eBGP报文"AS2 AS3 3c", 最后1c通过iBGP报文"AS2 AS3 3c"告知AS1的所有路由器.

### 确定最好的路由

在BGP通告前缀中有BGP属性, 其中比较重要的是 **AS-PATH** 和 **NEXT-HOP**. **AS-PATH** 记录了该BGP通告经过的AS列表, 如果一台BGP路由器在路径列表中发现了自己的AS, 他将拒绝该通告. **NEXT-HOP** 是 **AS-PATH** 起始路由器接口的IP地址, 在下图中, NEXT-HOP 指向起始路由器的接口.

![1702253252997](/img/计算机网络网络层/1702253252997.png)

- 热土豆路由选择: 每个网关路由器将BGP通告发给开销最小的 NEXT-HOP, 这里的开销可以是链路跳数或时延等.
- 路由器选择算法: 对于给定的目的地前缀, 路由器本地学习了一个或多个到达该前缀的下一跳路由, 如果是有多个将使用如下规则进行选择:
  - 路由器被指派一个本地偏好值, 这可能是由网络管理员设置或从其他路由学习到的.
  - 选择最短的 AS-PATH 路由, 如果该规则是路由选择的唯一规则, 那么将使用距离向量算法(AS 的跳数)来选择最短.
  - 使用热土豆路由选择, 选择 NEXT-HOP 最近的路由器路由
  - 使用 BGP 标识符来选择路由

### IP 任播

IP 任播通常用于 DNS 中. 教材使用了 CDN 来作为例子(虽然实际中并不使用), 对于每个 CDN 的服务器, 可能分布于世界上的不同地理位置, 但是为它们分配一个相同的IP地址, 这样通过 BGP 就可以将用户的请求发送到距离用户最近的 CDN 服务器上. 但是这可能导致同一个用户的同一个 TCP 连接请求发送到不同的 CDN 服务器上, 比如说当连接过程中 BGP 的最优选择改变.

## SDN 控制平面

### 特征

- 基于流的转发: SDN控制的交换机在进行分组转发工作时, 能够基于运输层, 网络层或链路层首部中任意数量的首部字段值进行. 比如前面的 Openflow 1.0 可以基于 11 个不同的首部字段进行转发.
- 数据平面与控股之平面分离: 数据平面是由网络交换机组成, 交换机在流表中执行匹配加动作的规则, 控制平面是由服务器以及决定和管理交换机流表的软件组成.
- 网络控制功能: 控制平面由一个 SDN 控制器以及若干网络控制应用程序两部分组陈, 这里的一个控制器是逻辑上的, 通常是在多台服务器上实现的.
- 可编程的网络: 通过运行在控制平面的网络控制程序, 该网络是可编程的.

### 组成

- 对于网络控制应用程序层的接口: 允许网络应用层通过 API 读/写网络状态与流表.
- 网络范围状态管理层: 由 SDN 控制平面做出的最终控制决定.
- 通信层: SDN 控制器和受控网络设备之间的通信.

![1702256264076](/img/计算机网络网络层/1702256264076.png)

### OpenFlow 协议

- 配置. 该报文允许控制器查询并设置交换机的配置参数.
- 修改状态. 该报文由控制器所使用，以增加／删除或修改交换机流表中的表项, 并且设置交换机端口特性.
- 读状态. 该报文被控制器用千从交换机的流表和端口收集统计数据和计数器值.
- 发送分组. 该报文被控制器用于在受控交换机从特定的端口发送出 一个特定的报文.
  从受控交换机到控制器流动的重要报文有下列这些：
- 流删除. 该报文通知控制器已删除一个流表项，例如由于超时，或作为收到“修改状态＂报文的结果.
- 端口状态. 交换机用该报文向控制器通知端口状态的变化.
- 分组入. 一个分组到达交换机端口，并且不能与任何流表项匹配，那么这个分组将被发送给控制器进行额外处理. 匹配的分组也被发送给控制器，作为匹配时所采取的一个动作. “分组入“报文被用于将分组发送给控制器.
