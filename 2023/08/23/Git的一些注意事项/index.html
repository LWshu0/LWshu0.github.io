一些实用的Git小技巧/建议

关于提交信息

规范

对规范的说明

<type>(提交类型) 描述
feat 添加新功能
fix 修复bug
refactor 重构代码，既不是修bug也不是添feature
build 涉及构建的改动
docs 文档的变更
style 代码格式更改，不影响代码运行
perf 性能优化
test 添加测试
revert 回退
merge 合并代码
ci 涉及集成
ui 样式修改
chore 其他修改，如构建过程或辅助工具的变动

scope是可选的信息，用于说明 Commit 影响的范围，可以是文件名、模块名、功能名等。

description是对 Commit 的简短描述，一般不超过50个字符。建议使用中文，结尾不加标点。描述一定要简洁、清晰

body是可选的，对 Commit 的详细描述。

footer是可选的，对 Commit 的脚注 --> 要关闭的 issue。

关于忽略不重要的文件

.gitignore文件中声明一些不需要跟踪的文件或文件夹。*号是通配符，可以匹配任意(包括0个)字符or字符串。*可以匹配一层目录，**可以匹配任意的目录层级。#号是注释。下面举一些例子来说明：

补充

符号 描述
!感叹号 用于取反，取消匹配的效果。可以在特定模式前使用，表示不忽略该文件或目录。
?问号 匹配一个任意字符（不包括目录分隔符）。
[]方括号 用于指定字符范围。可以匹配方括号内的任意一个字符。
{}花括号 用于定义一组模式，以逗号分隔。用于扩展匹配模式。
()圆括号 用于在模式内分组。

使用正则表达式
*.pattern regexp，其中pattern是一个自定义的正则表达式字符串。
