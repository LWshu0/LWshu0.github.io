计算机网络与因特网

发表于2023-08-27|更新于2023-08-27|计算机网络课程学习笔记
|字数总计:1.5k|阅读时长:4分钟

因特网

  • 其一，我们能够描述因特网的具体构成，即构成因特网的基本硬件和软件组件
  • 其二，我们能够根据为分布式应用提供服务的联网基础设施来描述因特网

具体构成

因特网是一个遍及全世界计算设备(称为主机(host)或者端系统(end system))的计算机网络.

端系统通过通信链路(communication link)和分组交换机(packet switch)连接到一起

  • 通信链路的传输速率以比特/秒(bit/s或者bps)度量
  • 端到端传输信息需要将数据分段, 称为分组(packet)
  • 分组交换机两种最著名的类型是 路由器(router) 和 链路层交换机(link-layer switch)
  • 一个分组所经历的一系列通信链路和分组交换机称为通过该网络的 路径 (route 或 path )
  • 端系统, 分组交换机和其他因特网部件都要运行一系列协议 (protocol),这些协议控制因特网中信息的接收和发送, 主要有 TCP(Transmission Control Protocol传输控制协议) 和 IP(Internet Protocol, 网际协议)

服务描述

  • 涉及多个相互交换数据的端系统的应用程序，被称为分布式应用程序 (distributed application)
  • 两个端系统上的软件通过 套接字接口(socket interface) 进行数据通讯

协议

协议 (protocol) 定义了在两个或多个通信实体之间交换的报文的格式和顺序，以及报文发送和／或接收一条报文或其他事件所采取的动作

网络结构

网络边缘

端系统也成为主机(host), 分为客户(client)和服务器(server), 其中客户通常指PC, 智能手机等, 服务器通常是更为强大的机器

接入网和物理媒介

指将端系统物理连接到其边缘路由器的网络, 边缘路由是端系统到其他任何远程端系统的路径上的第一台路由器

接入网

家庭接入

如今流行的两种方式: 数字用户线(igital Subscriber Line, DSL) 和电缆

家庭的 DSL调制解调器得到数字数据后将其转换为高频音，以通过电话线传输给本地中心局；来自许多家庭的模拟信号在 DSLAM 处被转换回数字形式

家庭电话线同时承载了数据和传统的电话信号，它们用不同的频率进行编码:

  • 高速下行信道，位于 50kHz 到 1MHz 频段
  • 中速上行信道，位于 4kHz 到 50kHz 频段
  • 普通的双向电话信道，位千 0 到 4kHz 频段

这种方法使单根 DSL 线路看起来就像有 3 根单独的线路一样， 因此一个电话呼叫和一个因特网连接能够同时共享 DSL 链路(频分复用)

电缆因特网接入 (cable Internet access ) 利用了有线电视公司现有的有线电视基础设施, 如下图为 混合光纤同轴 ( Hybrid Fiber Coax , HFC) 系统

DSLAM 和 CMTS 是不对称的, 即上行速率和下行速率不同, 通常下行速率要高, 因为相比较起来我们一般是接收数据多而发送数据少

企业(和家庭)接入

使用局域网(LAN)技术将端系统连接到边缘路由器, 以太网是目前最流行的接入技术

广域无线接入

WiFi, LTE, 3G, 4G等

物理媒体

物理媒体可分为:

  • 导引型媒体: 光缆, 双绞铜线等
  • 非导引型媒体: 无线局域网, 数字卫星通道等

具体描述:

  • 双绞铜线: 最便宜, 最常用, 一直用于电话网. 传输速率取决于线的粗细和传输方和接收方的距离
  • 同轴电缆: 俩个同轴的导体, 在电视中应用广泛, 一个发送方可对应多个接收方
  • 光纤: 传输速率极高, 不受电磁干扰, 成本高, 一般用于远距离传输(跨海链路)
  • 陆地无线电通道: 不需安装物理线路, 可穿墙, 受传播环境影响(障碍物, 干扰对象等)
  • 卫星无线电通道: 分为同步卫星, 近地轨道卫星, 需要地面站(与卫星相连)

网络核心

分组交换

将数据分为较小的数据块, 称为分组(packet). 每个分组通过通信链路和分组交换机(路由器, 链路层交换机)传送, 在两个路由器(或交换机)之间的传输时延:

one−hop transmission delay(sec)=L(bit)R(bit/s)one-hop\ transmission\ delay(sec) = \frac{L(bit)}{R(bit/s)} one−hop transmission delay(sec)=R(bit/s)L(bit)​

存储转发

交换机能够向链路传输该分组之前必须接收到整个分组(即先存储), 当全部收到后才开始向链路输出(即转发)该分组

那么在具有NNN条链路的一条路径上, 端到端的时延为:

dhost to host=NLRd_{host\ to\ host}=N\frac{L}{R} dhost to host​=NRL​

排队时延和分组丢失

每一台分组交换机有许多链路与之相连, 对于每一条链路都有一个输出缓存(output buffer, 也叫做输出队列(output queue)). 到达的数据需要先进入队列排队, 这就会产生排队时延(queuing delay), 排队时延取决于当前网络中的拥塞程度. 当当前队列充满后, 后续到达的数据将被丢弃, 也就是分组丢失(丢包)(packet loss).

转发表和路由选择协议

每一台路由器都有一个转发表, 用于将目的地址映射到输出链路, 也就是为分组选择接下来的链路, 通过一个个路由器的指引, 分组将最终到达目的地址.

路由选择协议用于自动的设置转发表.
